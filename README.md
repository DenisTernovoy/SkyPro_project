# Проект SkyPro_Widget

## Описание:

Проект SkyPro_Widget - это проект по созданию виджета банковских операций клиента.

## Установка:

1. Клонируйте репозиторий:
```
git clone https://github.com/DenisTernovoy/SkyPro_project.git
```

2. Выберите *Poetry* в качестве интерпритатора.

## Инструкции по работе с разработанными функциями

* Функция `get_mask_card_number` принимает номер банковской карты в формате `int` и возвращает замаскированный номер.

Пример:
```
print(get_mask_card_number(7000792289606361))
```

Результат работы функции:
```
7000 79** **** 6361
```
* Функция `get_mask_account` принимает номер банковского счета в формате `int` и возвращает замаскированный номер.

Пример:
```
print(get_mask_account(73654108430135874305))
```

Результат работы функции:
```
**4305
```
* Функция `mask_account_card` принимает информацию по банковским реквизитам в формате `str` и возвращает информацию с замаскированным номером.

Пример:
```
print(mask_account_card("Счет 35383033474447895560"))
print(mask_account_card("Visa Platinum 8990922113665229"))
```

Результат работы функции:
```
Счет **5560
Visa Platinum 8990 92** **** 5229
```
* Функция `get_date` принимает дату в формате `str` и возвращает дату в формате `dd.mm.YYYY`.

Пример:
```
print(get_date("2024-03-11T02:26:18.671407"))
```

Результат работы функции:
```
11.03.2024
```
* Функция `filter_by_state` принимает cписок словарей в формате `list[dict]` и возвращает отфильтрованный список словарей по необязательному параметру `state`.

Пример:
```
print(
        filter_by_state(
            [
                {"id": 41428829, "state": "EXECUTED", "date": "2019-07-03T18:35:29.512364"},
                {"id": 939719570, "state": "EXECUTED", "date": "2018-06-30T02:08:58.425572"},
                {"id": 594226727, "state": "CANCELED", "date": "2018-09-12T21:27:25.241689"},
                {"id": 615064591, "state": "CANCELED", "date": "2018-10-14T08:21:33.419441"},
            ]
        )
    )
```

Результат работы функции:
```
[{'id': 41428829, 'state': 'EXECUTED', 'date': '2019-07-03T18:35:29.512364'}, {'id': 939719570, 'state': 'EXECUTED', 'date': '2018-06-30T02:08:58.425572'}]
```
* Функция `sort_by_date` принимает cписок словарей в формате `list[dict]` и возвращает отсортированный список словарей по дате с учетом параметра, задающего порядок сортировки.

Пример:
```
print(
        sort_by_date(
            [
                {"id": 41428829, "state": "EXECUTED", "date": "2019-07-03T18:35:29.512364"},
                {"id": 939719570, "state": "EXECUTED", "date": "2018-06-30T02:08:58.425572"},
                {"id": 594226727, "state": "CANCELED", "date": "2018-09-12T21:27:25.241689"},
                {"id": 615064591, "state": "CANCELED", "date": "2018-10-14T08:21:33.419441"},
            ]
        )
    )
```

Результат работы функции:
```
[{'id': 41428829, 'state': 'EXECUTED', 'date': '2019-07-03T18:35:29.512364'}, {'id': 615064591, 'state': 'CANCELED', 'date': '2018-10-14T08:21:33.419441'}, {'id': 594226727, 'state': 'CANCELED', 'date': '2018-09-12T21:27:25.241689'}, {'id': 939719570, 'state': 'EXECUTED', 'date': '2018-06-30T02:08:58.425572'}]
```

* Функция `filter_by_currency` принимает cписок словарей в формате `list[dict]` и возвращает итератор, который поочередно выдает транзакции, где валюта операции соответствует заданной (например, USD).

Пример:

for _ in range(2):
    print(next(usd_transactions))

* Функция-генератор `transaction_descriptions` поочередно возвращает описание банковских транзакций.

Пример:
```
descriptions = transaction_descriptions(transactions)
for _ in range(5):
    print(next(descriptions))
```

Результат работы функции:
```
>>> Перевод организации
    Перевод со счета на счет
    Перевод со счета на счет
    Перевод с карты на карту
    Перевод организации
```

* Функция-генератор `card_number_generator` поочередно возвращает номера банковских карт в диапазоне от start до end.

Пример:
```
for card_number in card_number_generator(1, 5):
    print(card_number)
```

Результат работы функции:

## Тестирование

Для тестирования функций используется фреймворк `pytest`.
В пакете `tests` созданы тесты для каждого отдельного модуля проекта, что позволило обеспечить 100% покрытие функционального кода. 

С результатами тестирования можно ознакомится в [документе](htmlcov/index.html).
